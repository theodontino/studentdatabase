# 领域模型

Student Track 用于校外教学机构围绕学生生成和追踪课后家校反馈。系统会记录学生、课次、课堂评价、考勤、关键事件和家校沟通，但产品价值不只是记录，而是降低自然语言录入、沟通信息整理和反馈草稿生成的摩擦。领域术语以本文件和 `prisma/schema.prisma` 为准；页面文案可以更口语化，但不应创造含义相近的新概念。

## 核心术语

| 中文术语 | 代码实体 | 含义与边界 |
|---|---|---|
| 班级 | `Class` | 稳定的教学分组。`code` 是唯一业务编号，`name` 是展示名称。 |
| 学生 | `Student` | 学生基础档案。必须属于一个班级，`studentId` 是唯一学号。 |
| 学期 | `Semester` | 一段教学周期，包含起止日期和多个课次。 |
| 课次 | `ClassSession` | 一次实际教学活动。属于一个学期；可属于一个班级，`classId = null` 表示全校课次。 |
| 课次评价 | `SessionMetric` | 某学生在某课次的 A/B/C/D 四维评价，不是考试成绩总表。 |
| 考勤 | `Attendance` | 某学生在某课次是否出勤。每个学生、课次最多一条。 |
| 关键事件 | `Event` | 课堂表现、测验表现、心理状态等值得长期保留的事实。 |
| 家校沟通 | `Communication` | 围绕学生和课次发生的沟通摘要；外部导入可用 `sourceKey` 保存稳定去重身份。 |
| 录入草案 | `DraftRecord` | NL 解析后等待教师确认的临时业务草案。 |
| 评分变更历史 | `SessionMetricHistory` | 课次评价更新或删除前的不可变快照。 |
| 页面工作历史 | `WorkHistory` | 手动评分、录入、报告、导出等页面状态快照，不是学生档案。 |
| 系统日志 | `SystemLog` | 用于追踪关键系统操作，不参与业务计算。 |

## 评分语义

| 维度 | 含义 | 数据来源 |
|---|---|---|
| A | 学习目标达成与课堂测验表现 | 教师或经教师复核的 LLM 结果 |
| B | 精神面貌与课堂纪律 | 教师或经教师复核的 LLM 结果 |
| C | 课后任务 | 教师或经教师复核的 LLM 结果 |
| D | 考勤表现 | 系统根据当前学期考勤计算 |

四个维度当前均使用 0 到 5 的整数。`SessionMetric` 应优先绑定课次；`sessionId = null` 只用于兼容无课次来源，不应成为新功能的默认写入方式。

学生档案中的“学期综合分”是从原始记录实时计算的派生视图，不是考试成绩或新的持久化实体。A/B/C 分别取所选学期已发生课次的评价平均值；D 根据该学生在所选学期已有的明确考勤记录计算。缺少考勤记录不等于缺勤，未来课次不参与统计。四项完整时相加得到 20 分制综合分，再换算为百分制；任一数据类别缺失时显示数据不足，不写入或猜测总分。

## 领域约束

- 学生必须属于已存在的班级。
- 课次必须属于已存在的学期；班级课次只能评价该班学生，全校课次可以覆盖所有学生。
- 同一学生、同一课次最多一条课次评价和一条考勤。
- A/B/C 由教师确认，D 由系统计算；LLM 不直接决定 D。
- NL 录入选择班级课次后，花名册中未被提及的学生按缺勤处理。
- 课次评价被更新或删除前，应追加 `SessionMetricHistory`，不能覆盖旧历史。
- 页面工作历史恢复只改变页面状态，不直接写入学生、评价、事件或考勤。
- 日志、页面历史等辅助写入失败，不应把已经完成的核心业务写入显示为失败。
- 企微自动导入只有在会话唯一匹配学生、来源消息属于当前批次、学生与课次边界均通过确定性校验时才能写入；否则进入待复核或失败状态。
- 企微时间水位只缩小扫描范围，已处理判断以“会话 ID + 消息 ID + 内容指纹”的消息回执为准。

## 暂不新增的概念

`Exam` 目前不是独立领域实体。“测验”可记录为 A 维评价的一部分或 `Event`。只有当测验具有独立名称、日期、满分、参与名单、成绩集合和修改流程时，才论证新增 `Exam`。

`Score` 也不是独立实体。代码和文档应使用“课次评价”指代 `SessionMetric`，使用“维度分数”指代其中的 A/B/C/D 字段。

## 辅助模型

`Label`、`StudentLabel` 用于学生分类；`DraftRecord`、`WorkHistory`、`SystemLog` 和 `SessionMetricHistory` 支持业务流程与追踪，但不改变学生、班级、学期、课次、评价和考勤构成的核心领域。

`WeComImportState` 保存成功水位和运行租约；`WeComImportRun` 表示一次“一键同步并导入”；`WeComImportOperation` 表示运行中的单个会话批次；`WeComMessageReceipt` 区分未处理、处理中、已写入、无有效内容、待复核、失败、忽略和已回滚消息；`WeComImportChange` 只记录本批新增实体和标签贡献，不复制原始聊天正文或整库快照。这些模型属于集成处理账本，不是学生档案。

`AI内部关注：*` 是教师内部使用的全局学生标签。已确认的课堂文字可以添加“成绩表现、学习信心、家长担心、退班意向”四类受控标签；企微自动导入不创建这些标签。多个内部标签在仪表盘中只构成一个独立的“定性反馈”条件；它们不得进入家长反馈提示词、预览或导出，也不参与任何评分计算。

仪表盘的学生风险是实时派生视图，不写入数据库：前四次课以早期相对表现作为一个数字条件；第五次课起改用近期评分回落和长期低于同期班均。近期评分回落以最后 20% 有效课次和本学期个人总体平均之间的差距判定。一个独立条件显示“关注”，两个及以上显示“警告”。考勤提醒学生只在考勤区展示，不再进入上方学习风险排名；班级状态仍独立展示。

## 反馈上下文

课后反馈上下文不是新表，而是从现有领域模型临时组装出来的视图。它可以包含：

- 本课次的 `SessionMetric`、`Attendance` 和 `Event`。
- 近期若干课次的评价趋势和关键事件。
- 最近的 `Communication`，包括企微导入的长期家校沟通摘要。
- 学生已有标签。

反馈上下文只服务于生成和预览，不直接写回数据库。若上下文中出现错误，应修正原始学生记录、课次评价、事件、沟通或标签，而不是为上下文本身新增修补字段。
