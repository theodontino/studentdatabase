# 设计决策

本文件只记录影响长期维护、且无法仅从代码中看出原因的选择。

## SQLite 与 Prisma

当前使用 SQLite 和 Prisma。产品处于单人、本地优先验证阶段，SQLite 已满足数据量与并发需求；Prisma 提供迁移、类型和统一数据访问。只有出现明确的多用户并发、远程部署或 SQLite 限制时，才评估 PostgreSQL。

## 只支持本机访问

Web 服务显式绑定 `127.0.0.1`。当前产品不面向局域网或公网用户，因此不引入认证和远程部署基础设施。如果未来要求远程访问，必须先单独设计身份认证、授权、网络边界和敏感操作保护。

## WeComCatch 保持仓库外独立

Chem-Track 不包含或分发 WeComCatch 源码、运行数据及本地配置。可选集成只能通过 `WECOMCATCH_PROJECT_ROOT` 或 `WECOMCATCH_CLI_PATH` 指向仓库外的本地工具，并继续使用独立 SQLite 数据模型。公开 Chem-Track 不代表公开、授权或推荐该外部工具；使用者应自行确认平台协议、组织授权和个人信息处理依据。

## 领域模型优先于通用模型

系统以 `Student`、`Class`、`Semester`、`ClassSession`、`SessionMetric` 和 `Attendance` 为核心，不提前建立通用“内容”“任务”“考试”“分数”抽象。新表必须对应具有独立生命周期的业务概念。

## 渐进式 Service 层

不一次性为所有表建立 Repository 和 Service。简单 CRUD 保持在 Route Handler；出现多表事务、复杂规则或多入口复用时再抽离 Service。这样既形成清晰边界，也避免样板代码成为主要维护成本。

## 规则使用代码配置

可变化的预警阈值、评分常量和计算参数先放在 `src/config/` 的类型化配置中，不引入数据库配置中心或管理后台。只有实际出现教师自行配置需求后，才增加持久化设置。

## LLM 结果必须复核

自然语言解析和反馈生成允许使用 LLM，但学生身份、班级范围、考勤补齐和数据库写入由确定性代码控制。NL 解析结果先进入草案，经教师复核后才能成为正式业务数据。

课后反馈采用受控的两阶段模型流程：起草模型先写初稿，审核模型独立对照同一份确定性上下文，只能通过、修订或标记为需要人工确认，不得补写上下文中没有的事实。审核输出无法解析或疑似混入其他学生时，保留初稿并阻止导出，直到教师修改或重新生成。审核模型无法发现源记录本身错误，因此生成前上下文预览和最终人工检查仍然保留。

企微一键导入是受限例外：教师在启动前统一确认，只有会话唯一命中学生、模型学生 ID 位于服务端候选白名单、支撑消息 ID 属于当前会话批次、置信度为高且沟通日期能确定性映射到近 30 天内同班课次时，整批候选才可自动写入。任何条件不满足都不做部分写入，而是保留为待复核。这个例外不适用于 NL 评分、考勤或其他核心教学记录。

## 企微增量处理使用水位与消息回执

不要求教师维护永久的“企微会话—学生”人工绑定。会话标题只作为本地候选检索，必须唯一命中学生；时间水位只负责缩小扫描范围，不能单独证明消息已处理。正确性由“会话 ID + 消息 ID + 内容指纹”、处理状态和稳定来源键共同保证。一次点击建立运行租约，避免并发重复执行；过期租约和提取中回执可恢复。

企微提取的最小处理单位是连续交流段，而不是整个联系人历史或任意固定行数。上海自然日变化或消息间隔超过 6 小时开始新段，随后再按消息数和字符数硬拆分；这样既保留近邻语境，又让批次键、失败次数和回执能稳定对应一组不重叠消息。时间水位不替代消息回执，模型是否读过仍由回执与操作账本判定。

## LLM 原始交互使用本机操作缓存

模型输入、原始输出和推理内容可能是排障所必需，但不适合进入长期 Prisma 账本。它们按任务类型和一次用户操作保存在 Git 忽略的 `data/llm-cache/`，权限限制为当前用户；数据库只保存模型名、提示词版本、结束原因、token 数量、响应字符数和错误分类等非敏感诊断。某类任务下一次完整成功后清除该类更早缓存，失败只保留当天；跨日和总容量规则进一步限制留存。反馈成功不能删除企微失败缓存，反之亦然。

## 企微回滚使用增量日志

企微导入不为每个批次长期保存整库快照。一次点击下的批次在各自事务中记录新增沟通 ID、标签创建和标签贡献，可按运行、批次或日期回滚；仍被其他有效批次支持的标签不得删除。增量日志保留 30 天或最近 30 次运行。由于回滚本身是破坏性删除，执行前仍创建并校验一个短期 SQLite 安全备份，最多保留 3 份，以满足数据库安全规则。

## 录入与反馈共用课后工作台

自然语言录入、草案复核、反馈生成和导出属于同一次课后任务，因此在一个可恢复的五步工作台中组合。跨课次的待复核草案放在工作历史，避免打断当前课次。旧路由、`WorkHistory` 模块值和草稿存储键在 1.0 前保持兼容。

## 音频转写优先纯文本

Chem-Track 的课堂录音转写优先生成可复核的纯文本，而不是追求说话人分离、时间轴或会议纪要结构。当前业务目标是把课后评价内容更快送入 NL 录入；说话人标签和时间轴会增加阅读噪音，也会提高本地模型配置和故障排查成本。

本地转写使用 FunASR/SeACo 与热词增强。热词来自基础化学词表和当前学生姓名，属于输入质量增强，不改变学生身份匹配和业务写入规则。只有出现明确的课堂互动分析需求时，才重新评估说话人分离和时间轴。

## 历史分层保存

评价历史、页面工作历史、系统日志和数据库备份解决不同问题，因此分别保存。页面状态恢复不直接写业务表；数据库恢复只能使用通过完整性与校验和检查的备份。

## 暂不建立学期选课表

当前不新增 `Enrollment`。仪表盘按学生在目标学期课次中的考勤、评价、事件或沟通记录推导参与关系，班级归属使用该学期最近参与的课次。这满足当前班级按学期运行的方式，也避免为尚未出现的复杂转班流程增加维护成本。

如果旧学生长期留在原班级且新学期创建课次时仍被加入考勤，系统会将其视为新学期参与者。只有这种情况成为稳定业务需求，或需要准确记录同一学生跨学期转班、退班和复课时，才评估新增具有独立生命周期的 `Enrollment`。

## 文档只保留稳定认知

长期文档不记录版本进度、文件数量、测试数量和待办。机械事实由脚本生成，变化过程使用 GitHub Issues，减少每次改代码时同步多份文档的负担。

## SheetJS 使用官方发布源

Excel 导入导出继续使用 SheetJS Community Edition，但依赖指向 SheetJS 官方 CDN 的固定版本 tarball。npm registry 中的 `xlsx` 长期停留在旧版本并包含已知安全告警；固定官方 tarball既保留现有 Excel API 和格式兼容，也避免为了消除告警更换整套导入导出实现。升级前必须运行花名册导入、反馈导出和五 Sheet 数据导出回归测试。

## 使用 AGPL-3.0-only

Chem-Track AI 以 `AGPL-3.0-only` 发布。允许个人和组织使用、修改、分发及商业运营，但修改版被分发或用于向用户提供网络服务时，必须向相应用户提供完整对应源代码并继续使用同一许可证。选择 `only` 而不是 `or-later`，避免未来 GNU 许可证版本自动改变授权范围。

许可证只覆盖仓库中的软件和文档，不授权公开学生数据、运行数据库、音频、导出、日志或用户生成内容。theodontino 保留版权，并可在拥有完整版权的范围内另行提供商业许可证。

## 浏览器本地配色而非业务配置

界面提供经典、暮蓝、星云和平衡星云四套配色，默认使用平衡星云。配色只改变语义 token，并保存在当前浏览器 `localStorage`；不增加数据库字段、服务端设置或用户体系。页面首次绘制前由根布局应用已保存配色，避免深浅主题闪烁。

动效使用 CSS 和浏览器原生能力，不增加动画状态库。内容区使用较厚、较不透明的磨砂材质，液态玻璃感只用于导航和浮层；主题与动效都必须尊重可访问性、对比度和 `prefers-reduced-motion`。
