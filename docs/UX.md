# 用户体验与界面规范

## 产品气质

Chem-Track 是教师长期使用的本地教学工作台。界面采用冷静、专业、有温度的蓝色视觉：高频编辑页保持紧凑，档案与分析页适当留白。视觉升级服务于任务效率和数据理解，不使用装饰性玻璃拟态、霓虹、无意义渐变或持续动画。

界面设计借鉴成熟管理后台的共同原则：导航位置稳定、信息层级清楚、表单错误靠近输入项、操作结果可见、键盘和窄窗口可用。不得直接复制其他产品的品牌外观。

## 设计基础

- 只使用系统字体，不依赖在线字体资源。
- 正文使用 14px 或 16px，辅助信息使用 12px；页面、区块和控件标题必须形成稳定层级。
- 间距只从 4、8、12、16、24、32px 阶梯中选择，避免一次性数值。
- 圆角只使用 8、12、16px；阴影只区分静态表面和浮层两档。
- 蓝色只承担品牌、链接、选中和主要操作；绿色、琥珀色、红色分别承担成功、警告和危险。
- 状态不能只依靠颜色表达，必须同时具有文字、图标或可访问名称。
- 单个操作区只突出一个主要操作；危险操作使用明确文案并二次确认。
- 过渡时长控制在 120–200ms，并尊重 `prefers-reduced-motion`。
- Emoji 不作为正式导航或操作图标。

颜色、间距、圆角和阴影必须引用 `src/app/globals.css` 中的语义 token。业务组件不得新增同义颜色常量或复制一套局部主题。

## 五种页面模板

每个页面必须采用以下一种模板，不新增第六套页面骨架：

1. **概览页**：仪表盘。先显示范围与摘要，再显示趋势和需要行动的事项。
2. **高频工作台**：快速评分、课堂录入、课后反馈、班级日报和转写。上下文与主要操作稳定可见，编辑内容优先于装饰。
3. **列表管理页**：学生、学期、工作历史和导出。由页面标题、筛选工具栏、结果区和明确的空状态组成。
4. **档案详情页**：学生和学期详情。身份摘要保持稳定，分析与历史记录分区展示。
5. **系统页**：配置、集成和维护。共享系统子导航，不合并为一个巨大客户端页面。

页面宽度按工作类型选择：概览与详情使用舒展的受限宽度；多列编辑工作台可以使用更宽画布；正文不得无理由贴满超宽屏。

## 教学上下文与页面连续性

- 学期、班级和课次以 URL 参数为事实来源，页面之间跳转时继续携带。
- 学期变化清空班级和课次，班级变化清空课次。
- 可恢复的筛选、草稿和工作状态使用带版本与运行时校验的标签页级 `sessionStorage`。
- 涉及教学数据的草稿必须按页面、学期、班级和课次隔离。
- 页面恢复后显示已恢复状态；不得把另一个课次的草稿静默带入当前页面。
- API Key、Token、File、Blob、音频和凭据不得写入页面恢复数据。
- 无法恢复的本地文件在离页前必须提醒。

## 操作与状态反馈

加载、空数据、部分数据、错误、成功和危险操作使用共享 UI 组件。保存型工作流统一表达以下状态：

```text
未修改 → 有修改 → 保存中 → 已保存
                  ↘ 保存失败（保留修改并允许重试）
```

AI 工作流统一表达以下状态：

```text
等待 → 校验 → 生成 → 待复核 → 保存 → 完成
                ↘ 失败 / 已取消（保留安全输入并允许恢复）
```

不得使用假进度百分比。外部 LLM 或本地工具失败不能让已经成功保存的业务数据显示为失败。

## 表单、浮层与数据展示

- 表单项统一包含 label、可选说明、错误和必填状态；错误显示在对应字段附近。
- Dialog 用于短任务，Drawer 用于不离开当前上下文的详情或较长任务。
- Dialog 和 Drawer 必须支持 Escape、焦点进入与返回、背景滚动锁定和可访问标题。
- 表格和列表必须有加载、空数据与失败状态；操作列不能只显示含义不明的图标。
- 图表必须说明时间范围、样本数量和缺失数据，不能把缺失值画成零。
- 综合分、预警和 AI 结论必须展示足以解释结果的上下文，不把颜色当作结论。

## 响应式与可访问性

- 以桌面为主要编辑环境，在 900px 以下切换为抽屉导航。
- 720px 宽度必须能够完成导航、查看档案和核心教学工作流。
- 不承诺手机宽度下的完整批量编辑，但不得出现无法关闭的浮层或横向页面整体溢出。
- 所有交互控件必须有键盘焦点和可访问名称；点击区域不得小于约 36px。
- 窄窗口不得通过隐藏主要操作来解决布局问题，应改为换行、纵向排列或固定操作栏。

## 代码约束

- Route 页面只组合 Feature；展示组件不承载复杂请求、SSE、历史恢复或业务规则。
- 通用 UI 不依赖业务 Feature。跨 Feature 复用只能通过公开导出。
- 不新增 `any`，不建立第二套 Student、Semester、Session、Draft 或 Feedback 类型。
- 大型页面按真实职责逐步拆分，不为了行数制造无意义文件。
- 新 UI 必须复用语义 token 和共享组件。一次性 Tailwind 样式只用于明确的业务布局，不复制按钮、输入框、状态或浮层外观。
- 兼容路由、WorkHistory、sessionStorage 键、API、SSE/NDJSON 和 Excel 格式在 1.0 前保持兼容。
