# 隐私化开发方针

Chem-Track 处理未成年学生的教学记录，代码仓库默认按“未来可能公开”设计。任何提交都必须满足：没有真实业务数据、没有凭据、没有个人设备信息，且测试数据可明确识别为合成数据。

## 禁止进入 Git

- 真实学生姓名、学号、联系方式、班级名单、评价、考勤、事件、家校沟通和反馈。
- SQLite 数据库、备份、Excel/CSV 导出、音频、转写结果、截图、日志、诊断文件和运行目录。
- `.env`、API Key、Token、Cookie、私钥、本地 LLM 凭据和第三方服务配置值。
- 个人绝对路径、设备名、本机邮箱、内网地址、个人工作区位置和真实任务运行统计。
- 从外部工具复制来的 handoff、聊天记录、临时计划或含运行状态的排障文档。

`.env.example` 只能包含显式占位符。任何需要保留的示例必须使用固定合成 fixture，并以 `test-`、`example-` 等前缀标识；中文示例姓名只使用“张三、李四、王五”等通用占位名。

## 代码与运行要求

- 路径使用项目相对路径、`$HOME`、`os.homedir()` 或显式环境变量，禁止写入个人主目录。
- 服务端日志只记录操作类别、内部 ID 和安全摘要，不记录学生原文、反馈全文、文件内容或凭据。
- 错误响应不得返回环境变量、绝对路径、SQL 内容或第三方原始响应。
- 测试只能使用隔离临时数据库；不得复制、抽样或匿名化真实 `dev.db` 作为 fixture。
- 向 LLM、ASR 或其他外部服务发送数据前，UI 必须说明可能离开本机；自动化测试必须阻断真实外部请求。
- 导入、导出和备份文件仅保存在被 Git 忽略的运行目录。
- 页面恢复数据只能使用标签页级 `sessionStorage`、带版本键和运行时类型校验；不得写入 API Key、Token、未上传音频、导入文件或其他凭据。
- 页面恢复是本机标签页内的临时便利功能，不得同步到云端、日志或 Git；需要长期保留的工作状态使用经用户操作生成的 `WorkHistory`。

## 提交与发布闸门

每次提交前运行：

```bash
npm run privacy:check
git status --short
```

公开仓库或发布新版本前还必须：

1. 扫描当前树和完整 Git 历史中的密钥、数据库、导出文件和个人路径。
2. 复核 Git 作者姓名和邮箱是否适合公开。
3. 确认真实数据库和运行目录未被测试改变。
4. 若历史曾包含禁止内容，先备份 refs，再重写历史并重新扫描；不得只删除当前文件。

## 泄露处置

发现疑似泄露后立即停止推送和公开发布。凭据先撤销或轮换，再清理历史；业务数据先确认影响范围和远端副本，再决定通知与删除方案。禁止仅用后续提交“删除”敏感文件，因为旧内容仍可从 Git 历史恢复。
