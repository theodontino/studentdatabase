# 架构设计

## 目标

Chem-Track 是单人维护的本地优先课后反馈自动化工具。架构优先保证数据安全、业务规则可读和低维护成本，不为尚未出现的并发或组织规模预建基础设施。Web 服务只绑定 `127.0.0.1`；当前不支持局域网或公网部署，也不以缺少身份认证的状态对外暴露。

## 前端模块边界

App Router 页面只组合 Feature 工作台或执行兼容重定向。交互状态、请求和历史恢复归属 `src/features/`；通用控件归属 `src/components/ui/`，HTTP JSON 与文件下载归属 `src/lib/api-client.ts`。Feature 通过公开入口复用共享能力，不直接引用另一个 Feature 的内部文件。

学期、班级、课次采用显式 URL 上下文：`semesterId`、`class`、`sessionCode`。学期变化清空班级和课次，班级变化清空课次。URL 是当前页面的事实来源；标签页级 `sessionStorage` 只记录最近教学上下文，用于在工作台之间导航时补全 URL，不作为全局数据缓存。

未提交的页面工作状态使用带版本包装的标签页级 `sessionStorage` 保留，涉及教学数据的工作台按“页面 + 学期 + 班级 + 课次”隔离，避免把旧草稿恢复到另一课次。工作台只能通过 `src/lib/session-workspace.ts` 和对应 hook 读写，必须校验恢复数据，版本不匹配时放弃恢复。该存储用于切换页面和刷新后继续当前工作，不替代数据库 `WorkHistory`。凭据、API Key、未提交的 `File`/`Blob` 和录音流不得写入页面工作状态；浏览器无法安全恢复的本地文件必须在离页前提醒。已创建的转写任务只保留任务 ID，结果仍从服务端任务目录读取。

展示型 TSX 由 Playwright 覆盖关键工作流和窄屏导航；Vitest 的显式统计范围为 `src/app/api/**/*.ts`、`src/services/**/*.ts`、`src/lib/**/*.ts` 和 `src/features/**/*.ts`，并排除展示型 TSX、纯类型与入口文件。这样新增但尚未被测试加载的逻辑也会进入统计，页面从 `app/` 迁入 Feature 时不会改变覆盖率口径。v0.19.0 的新口径基线为 Statements 40.67%、Branches 42.58%、Functions 44.79%、Lines 44.98%；v0.19.1 为 42.53% / 43.96% / 47.93% / 46.52%；v0.20.0 为 42.46% / 43.81% / 48.01% / 46.49%；v0.20.1 为 42.58% / 44.47% / 48.72% / 46.73%；v0.20.2 为 47.15% / 47.30% / 53.12% / 51.33%，未低于前版基线。

AI 任务通过 `src/features/ai-workflow/` 的判别联合与 reducer 表达校验、生成、复核、保存、完成、失败和取消。页面刷新后，仍处于运行阶段的存档必须恢复为可重试的中断状态，不能假装后台任务仍在继续。该状态只记录任务名称、阶段、时间和安全提示，不保存凭据、文件或模型请求正文。

快速评分的参考资料、课次协调和保存/历史职责分别由独立 hook 管理；页面组合 hook 不重复实现差异提交或 ref 协调。录音转写同样把浏览器录音、任务 API、任务列表和结果操作分离，服务端任务 ID 是跨页面恢复的唯一运行标识。

仪表盘的数字风险必须按单一学期、实际课次班级和同课次班均计算；未来课次、无课次评价和其他学期数据不得混入。前四次课的相对表现、第五次课后的近期评分回落与长期低于班均均由纯规则模块计算。近期评分回落比较最后 20% 有效课次与学生本学期个人总体平均，至少需要 5 次有效评价。已触发考勤提醒的学生不再进入上方学习风险候选和早期排名。全局内部关注标签只在当前自动解析学期作为一个定性条件参与叠加，不能改写历史学期结果。Dashboard 加载不得调用 LLM。

企微桥接和课堂录入共享受控的内部关注原因枚举。LLM 只生成候选；标签随既有正式写入事务落库，废弃草案和重复企微记录不得产生标签。所有家长反馈上下文必须先过滤 `AI内部关注：*`，不能依赖提示词要求模型自行忽略。

学生档案的学期四维平均和综合分由 Service 按请求批量聚合，列表和详情复用同一计算规则。该结果是可重建的展示数据，不写入数据库或浏览器缓存；修改评价、考勤或课次后，下次请求必须从原始记录得到最新结果。列表聚合禁止逐学生查询，只有真实规模验证超过既定性能阈值时才另行论证持久化缓存。

当前学期优先选择日期范围覆盖今天且 `startDate` 最新的学期；无匹配时回退到拥有最近课次的学期，再回退到开始日期最新的学期。学生是否参与某学期由该学期课次中的考勤、评价、事件或沟通记录推导，历史班级归属以该学期最近参与课次为准。

## 请求链路

```text
Page / Component
  -> Route Handler
    -> Service（复杂业务、事务、跨入口复用）
      -> Prisma
        -> SQLite
```

- 页面负责交互和展示，不直接访问 Prisma，不承载业务判断。
- Route Handler 负责输入解析、HTTP 状态和响应格式。
- 简单单表 CRUD 可以直接使用 Prisma。
- 多表写入、复杂规则、事务或多入口复用进入 `src/services/`。
- Repository 只在复杂查询被多个 Service 重复使用时建立，不作为固定层级。
- 教师或机构可能调整的规则进入 `src/config/`；数据库约束与稳定领域不变量保留在 Schema 或 Service。

## 数据边界

`prisma/schema.prisma` 是数据库结构的唯一事实来源。Schema 变更必须通过 migration，并在全新数据库和现有数据升级路径上验证。

一次用户动作涉及多个核心写入时必须使用事务。例如确认录入草案时，评价、考勤、事件、沟通和草案状态要么共同成功，要么共同失败。操作日志和页面工作历史属于辅助数据，可以在核心事务成功后独立写入。

## LLM 边界

LLM 负责提出解析结果和文本建议，不是业务事实来源。姓名映射、班级边界、未提及学生缺勤、分数范围和最终写入由确定性代码及教师确认控制。

外部 LLM 失败只能使当前生成任务失败，不得修改已经保存的学生档案或历史记录。流式接口应使用统一 SSE 编解码工具，结束事件和错误事件必须可区分。

课后反馈生成必须先由 Service 组装确定性上下文，再交给 LLM 写作。上下文可包含今日课次表现、近期趋势、家校沟通和学生标签；LLM 只负责把这些上下文写成家长可读文本，不能决定学生身份、课次边界、考勤补全、D 分或数据库写入。

反馈生成 Service 先调用起草角色，再将同一份确定性上下文与初稿交给审核角色。审核结果是通过、已修订或需人工确认；审核 JSON 异常时不采纳未知文本，而是保留初稿并标记人工处理。两个角色在本地 LLM 配置中引用现有配置档案；未指定时都跟随当前激活档案，因此可以使用同一模型。密钥仍只保存在本机运行配置，不进入 Prisma 数据库。

企微结构化提取使用独立模型角色。协议协商只允许 JSON Schema 和 JSON Object：优先发送 Schema 并关闭推理；只有提供方不支持推理参数时才去掉该参数，只有不支持 Schema 时才降级到 JSON Object。两种结构化协议都不可用时任务停止，禁止退回普通文本。服务必须检查结束原因；输出截断不能原样重试，而应拆小当前交流段。

所有实际发起的 LLM 任务按一次用户操作写入 `data/llm-cache/<上海日期>/<任务类型>/<操作 ID>/`。缓存保存请求、模型原始响应、推理内容和安全诊断，但不保存请求头或凭据；它是本机短期排障材料，不替代业务数据库、消息回执、候选记录或增量回滚日志。缓存写入失败只能形成维护警告，不能改变业务事务结果；维护 API 只返回状态、次数、时间和大小等元数据。

外部企微数据进入 Chem-Track 时分为两步：先由 LLM 将导出文本整理为候选 JSON，再由确定性导入服务匹配学生、绑定课次、判断置信度、预览和写入。应用运行时不依赖 OpenClaw skill 才能完成核心导入流程。

WeComCatch 是仓库外的可选本地工具，不属于 Chem-Track 源码或发布物。Chem-Track 仅保留通过环境变量配置外部 CLI 的适配边界；未配置时相关功能不可用，核心业务不受影响。外部工具继续使用独立 SQLite 数据模型，不并入 Prisma。

一键企微导入仍通过该 CLI 边界工作：一次点击先取得数据库运行租约，再完成同步和导出。成功水位加短重叠窗口负责缩小扫描范围，“会话 ID + 消息 ID + 内容指纹”回执负责正确去重；没有原生消息 ID 时使用稳定内容指纹，不使用 JSONL 行号。标题只能唯一命中一个学生，否则不交给 LLM 自动处理。一个联系人会话按上海自然日或超过 6 小时的消息间隔划分连续交流段，再限制为最多 30 条消息或 8000 字符；段与段之间没有消息重叠。超过 8000 字符的单条消息独立处理，超过 20000 字符的单条消息直接进入人工复核。

LLM 每次只收到当前候选学生、当前交流段新增消息及该学生少量近期沟通摘要。模型返回的学生和来源消息必须再次落在服务端白名单中；自动课次只能绑定到沟通日期前 30 天内最近的同班课次，无法确定时整段保留候选并进入待复核，不做部分写入。每个通过校验的交流段在独立事务中写入 `Communication`、受控标签贡献、消息回执和增量变更日志；成功段立即提交，后续段失败不会撤销前段，但同一次运行可以整体回滚。稳定段自动失败两次后暂停，只有教师明确重新排队或更换提取模型后才再次调用 LLM。

回滚长期只保留新增实体 ID 与标签贡献，按 30 天或最近 30 次运行清理。执行删除前仍按通用数据安全规则创建并校验一次 SQLite 安全备份，自动只保留最近 3 份。进程中断后的过期租约、`extracting` 回执和失败候选可以恢复；浏览器断开只停止进度显示，不中断服务端事务。

## 音频转写边界

音频转写是 NL 录入的辅助输入，不是核心领域模型。转写任务、音频文件、日志和中间结果保存在 `data/diarize/`，不写入 Prisma 业务表；只有用户将转写文本送入 NL 录入并复核确认后，才可能产生正式的评价、考勤、事件或沟通记录。

Chem-Track 的本地转写入口通过项目根目录 `diarize.sh` 委托外部 FunASR 工具。项目入口负责选择 Chem-Track 适用的模型、纯转写模式和任务级热词；外部工具保留通用能力，不承担 Chem-Track 的学生名单或业务语义。

转写结果可用于提高录入效率，但不是事实来源。姓名匹配、未提及学生缺勤、课次边界和最终写入仍由 NL 解析后的确定性逻辑及教师复核控制。

## 历史与日志

- `SessionMetricHistory` 保存业务数据变更前快照，用于追溯评价。
- `WorkHistory` 保存可恢复的页面工作状态，由用户主动清理。
- `SystemLog` 保存关键操作摘要，失败不回滚核心业务。
- 数据库备份用于灾难恢复，不能由以上任一表替代。

## 文档边界

- `DOMAIN.md`：稳定术语、实体边界和不变量。
- `ARCHITECTURE.md`：分层、数据流和稳定工程约束。
- `OPERATIONS.md`：启动、迁移、备份、恢复和验证。
- `DECISIONS.md`：少数需要长期解释的重要选择。
- `generated/`：从代码生成的 Schema、路由和 ER 图，不手工维护。
- GitHub Issues：Feature、Bug、Refactor 和技术债等变化过程。
